description: Una vez que se estudió el sitio y sus funcionalidades estamos en condiciones de seleccionar una muestra de sus páginas internas que sea representativa para ser auditada.

Lo ideal es evaluar cada una de las páginas de un sitio. En casos puntuales, no se podrá porque son muchas las páginas a evaluar. Por ende, una buena estrategia es **tomar una muestra** de ella que permita obtener un **buen nivel de accesibilidad.**

Cuando no se puede realizar una evaluación del sitio completo, los **subpasos** para realizarla son:

  * **Incluir una muestra estructurada** de las páginas y estados de páginas que se identificaron en el [paso 2](https://gcba.github.io/estandares/pautas/accesibilidad/metodologia/explorar_el_sitio/)(explorar el sitio web).
  
      - Las páginas identificadas como relevantes
      - Las páginas que tienen funcionalidades esenciales
      - Diferentes tipos de páginas o estados de páginas
      - Las tecnologías de la que dependen los sitios

 > **Importante:** la cuidadosa selección de las páginas puede reducir significativamente el tamaño de la muestra requerida manteniendo una representación adecuada de todo el sitio web.

  * **Incluir procesos completos en la muestra:** se tienen que evaluar todas las páginas del proceso, no se pueden evaluar solo algunas. Recomendamos describir en detalle cómo se identifican y seleccionan las muestras.

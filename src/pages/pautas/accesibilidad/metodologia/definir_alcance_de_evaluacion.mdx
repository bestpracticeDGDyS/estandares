El alcance se debería acordar de forma conjunta con quienes se han encargado de la evaluación, y de este modo asegurar expectativas comunes. Se divide en los siguientes **subpasos**:


 > **Importante:** Algunos pasos pueden llevarse a cabo en paralelo pero es necesario tenerlos en cuenta a todos para garantizar una evaluación fiable.


## A) Definición de qué se va a evaluar

   - **Página web**
   
   - **Sitio web**
   
   - **Proceso completo:** secuencia de pasos que se necesitan realizar dentro de diferentes páginas web para completar una actividad. En esta evaluación, cada una de las páginas debe tener **el mismo nivel especificado.**
      
Para llevarla a cabo, se requiere **conocer** las propiedades de algunas partes del sitio. En simultáneo, aplicar el [Paso 2: Explorar el sitio web](https://gcba.github.io/estandares/pautas/accesibilidad/metodologia/explorar_el_sitio/).

## B) Definición del nivel de adecuación

Definir cuál es el nivel de conformidad (A, AA, AAA) que se va a evaluar.

 - **Nivel de Conformidad “A”**
    - Incluye todas las pautas (30).

- **Nivel de Conformidad “AA”** Incluye las siguientes 13 pautas:

    * [1.2.4 AA Subtítulos (directo)](https://www.w3.org/TR/UNDERSTANDING-WCAG20/media-equiv-real-time-captions.html)
    * [1.2.5 AA Audiodescripción (pregrabada)](https://www.w3.org/TR/UNDERSTANDING-WCAG20/media-equiv-audio-desc-only.html)
    * [1.4.3 AA Contraste mínimo](https://www.w3.org/TR/UNDERSTANDING-WCAG20/visual-audio-contrast-contrast.html)
    * [1.4.4 AA Cambio de tamaño del texto](https://www.w3.org/TR/UNDERSTANDING-WCAG20/visual-audio-contrast-scale.html)
    * [1.4.5 AA Imágenes de texto](https://www.w3.org/TR/UNDERSTANDING-WCAG20/visual-audio-contrast-text-presentation.html)
    * [2.4.5 AA Múltiples medios](https://www.w3.org/TR/UNDERSTANDING-WCAG20/navigation-mechanisms-refs.html)
    * [2.4.6 AA Encabezados y etiquetas](https://www.w3.org/TR/UNDERSTANDING-WCAG20/navigation-mechanisms-descriptive.html)
    * [2.4.7 AA Foco visible](https://www.w3.org/TR/UNDERSTANDING-WCAG20/navigation-mechanisms-focus-visible.html)
    * [3.1.2 AA Idioma de partes](https://www.w3.org/TR/UNDERSTANDING-WCAG20/meaning-doc-lang-id.html)
    * [3.2.3 AA Navegación consistente](https://www.w3.org/TR/UNDERSTANDING-WCAG20/consistent-behavior-unpredictable-change.html)
    * [3.2.4 AA Identificación consistente](https://www.w3.org/TR/UNDERSTANDING-WCAG20/consistent-behavior-consistent-locations.html)
    * [3.3.3 AA Sugerencia tras error](https://www.w3.org/TR/UNDERSTANDING-WCAG20/minimize-error-suggestions.html)
    * [3.3.4 AA Prevención de errores en páginas legales, financieras y de datos](https://www.w3.org/TR/UNDERSTANDING-WCAG20/minimize-error-reversible.html)

- **Nivel de Conformidad “AAA”** Incluye las siguientes 2 pautas:
    * [2.1.3 AAA Teclado (sin excepciones)](https://www.w3.org/TR/UNDERSTANDING-WCAG20/keyboard-operation-all-funcs.html)
    * [3.3.5 AAA Ayuda](https://www.w3.org/TR/UNDERSTANDING-WCAG20/minimize-error-context-help.html)
    
    [Accedé a todas las pautas](https://www.w3.org/TR/UNDERSTANDING-WCAG20/Overview.html) para poder realizar la evaluación.


## C) Definir el soporte de la accesibilidad

Definir el listado de navegadores web, productos de apoyo u otros agentes de usuario con los que deben ser compatibles:

   * **Herramientas automáticas:** nos permiten analizar el código de una página web, devolviendo como resultado un reporte con una serie de anotaciones con los criterios verificados exitosamente y con las fallas o problemas encontrados.
   
   * **Links de herramientas de evaluación:**
   
       - [Wave Evaluation Tool - Sitio del proyecto](https://wave.webaim.org/extension/)
       - [Web Developer Toolbar - Sitio del proyecto](https://chrispederick.com/work/web-developer/)
       - [Web Accessibility Toolbar - Sitio del proyecto](https://developer.paciellogroup.com/resources/wat/)
   
   * **Extensión para Mozilla Firefox:**
   
       - [WAVE Evaluation Tool](https://addons.mozilla.org/en-US/firefox/addon/wave-accessibility-tool/)
       - [Web Developer Toolbar](https://addons.mozilla.org/es/firefox/addon/web-developer/)
   
   * **Extensión para Google Chrome**
   
       - [WAVE Evaluation Tool](https://chrome.google.com/webstore/unsupported/jbbplnpkjmmeebjpijfedlgcdilocofh)
       - [Web Developer Toolbar](https://chrome.google.com/webstore/unsupported/bfbameneiokkgbdmiekhjnmfkcnldhhm)
   
   * **Extensión para MS Internet Explorer**
   
       - [Web Accessibility Toolbar](https://developer.paciellogroup.com/resources/wat/)
   
   * **Repositorio de código abierto:**
   
       - [Web Developer Toolbar](https://github.com/chrispederick/web-developer/)
       - [Web Accessibility Toolbar](https://github.com/ThePacielloGroup/WebAccessibilityToolbar/releases/tag/2015-05-20)
   

Aunque este tipo de evaluación automática se basa en herramientas muy útiles por su velocidad y simpleza, muchos aspectos de accesibilidad sólo pueden verificarse mediante una **revisión manual complementaria.**
    
   * **Herramientas manuales:** nos ayudan a verificar de **forma manual** cada uno de los criterios de conformidad web. Esta forma de verificación es un **complemento** necesario a la evaluación automática e implica la participación de un analista familiarizado con las WCAG 2.0. Este analista verificará cada criterio de conformidad pudiendo utilizar herramientas automáticas en línea y “extensiones” integradas al navegador, complementando su evaluación mediante la navegación de la página utilizando:
   
       * **Sólo el teclado**
       
       * **Diversos dispositivos** (escritorio, tablets, móviles- o una herramienta para emularlos)
       
       * **Diversos navegadores** (Firefox, Chrome, Explorer, Safari, entre otros)
       
       * **Lectores de pantalla:**
          - [NVDA](https://www.nvaccess.org/), lector de pantalla para Microsoft Windows gratuito.
          - [VoiceOver](https://www.apple.com/es/accessibility/mac/vision/), lector de pantalla disponible en el sistema operativo Mac OS X.
       
**Complementar la evaluación con pruebas que incluyan usuarios:** es recomendable realizar pruebas con personas con y sin discapacidad, que utilicen distintos tipos de apoyos (visuales, motrices, etc.) para navegar la web, y probar el nivel de accesibilidad.

Este paso permite comprender mejor el **uso**, **propósito** y **funcionalidad del sitio**. Además ayuda a identificar las páginas relevantes o con problemas. El evaluador debe poder acceder a todas las páginas.

   > **Importante:** Es más eficaz la evaluación si se involucran a los   
   desarrolladores y a los propietarios del sitio.

La exploración se divide en los siguiente **subpasos:**

   *   **Identificar las páginas (o estados de las páginas) relevantes:** por ejemplo, el inicio, la página de registro y otras páginas de entrada.

   *   **Identificar las funcionalidades claves del sitio:** es una lista de funcionalidades que los usuarios pueden realizar en el sitio. Por ejemplo, selección y compra de un producto.

   *   **Identificar los diferentes tipos de páginas y estados de página:** por ejemplo, el contenido web con diferentes estilos, diferentes componentes funcionales, diversas tecnologías, (HTML, CSS, JavaScript, WAI-ARIA, PDF) ; Contenido dinámico: mensajes de error, pop-ups; etc.

   *   **Identificar las tecnologías de las que se depende:** Pueden ser HTML, CSS, JavaScript, WAI-ARIA, SMIL, SVG, PDF, etc.
